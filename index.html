What we are building

Memory observability layer

Lightweight clients wrap the vector and key‑value stores your agents use as long‑term memory. We log each write and read, plus the tools and actions that follow, creating a clean timeline from memory to behavior.

Attribution & causality engine

Our backend correlates unexpected behaviors with specific memory chunks and surfaces patterns that look like memory poisoning, persistent backdoors, or goal drift—“this happened because of M‑324,” not just “something is off.”

Memory‑aware control plane

A simple console lets security and platform teams inspect influential memories, quarantine or mute suspicious chunks, and roll back to pre‑compromise state without touching model weights or application logic.

Why now

☣ Zombie agents

Agents now have long‑term memory and real system access (email, CRM, billing). When that memory is poisoned, they quietly replay attacker instructions across sessions, and nobody can explain where the behavior came from.

No visibility at the memory layer

Existing tools watch prompts, APIs, and permissions, but treat memory as an opaque blob. The decisions that scare CISOs most are often driven by stored state they never see.

Our wedge

Membrane owns the causal link between memory and action. We don’t replace your observability or security stack—we give it the missing signal: which memories are making your agents act strangely, and what to do about it.
